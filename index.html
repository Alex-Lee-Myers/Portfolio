Project Details for

GameChest

Technologies used:

Link's awakening.

GameChest was a project I worked on in a team of three over a period of two weeks. The project was to create a full-stack web application where unauthorized users would be able to see reviews of games and ratings of them. The application would also allow authorized users to create reviews and ratings of games including create a new game entry themselves. The application was built using React, Node.JS, Express, PostgreSQL, Material Design Bootstrap, React-Router-Dom, Bcrypt, JsonWebToken (JWT), Express, Reactstrap, and React-Burger-Menu.

Below you can read about the parts I held full responsibility over.

Login and Register

The User Experience

My original main responsibility was for the authorization of users for both the frontend and backend. I created login and register modals using React-Router-Dom and Reactstrap. In addition, I created a user model to store user information in the database that was supported with JWT and Brypt.

I learned in heavy detail how to properly prompt a user in both login and register with user interface details to ensure that the user had a correct experience. There was a plethora of nested if statements to ensure that the user was able to login and register properly and the site was secure.

Sidebar

Authorized Routes

Due to the time parameters placed on the project, I utilized an npm package called "React Burger Menu" to quickly create a sidebar that users would use to navigate through the application. With conditions based on useState's for authorization, the user would dynamically see various functional buttons between logout, creating a game, creating a review, and updating previous entries they had created.

All (App.js) the Rest

Home is Where the Teamwork Lay

With the aspect of being on heavy constraints for a shortened sprint, I took part in many hours-long debug sessions for all aspects of the application. This is where I found my furthest growth and in particular for how props get passed through multiple components and the case for state managers like Redux and Remix. In this, I learned useEffect, conditionals for returns on the frontend, as well as React-Router-Dom version 6.

For the latter, utilizing the useHistory hook to navigate to different pages as well as useNavigate hook to navigate to different pages. This made the user experience much more fluid and streamlined.

Project Details for

Location Station

Technologies used:

Locating the State of Things

Location Station was an intense 72-hour team project that was assigned in the midst of my bootcamp with Eleven Fifty Academy. It was not for the faint of heart due to not instructor help throughout the process, differing skill levels between teammates, and coming fresh off our main section of learning React, but it was a great experience. I was the git master and leader of a team of three originally. Due to the nature of the time instensivity, I ended up doing the vast majority of the work. I was grateful for this as it led to an exceptional learning breakthrough in how React works.

The project was to use React and build a site that incorporated the OpenWeather API, TicketMaster API, and NASA API. The site was to be a single page application that would allow users to grab their location and pull up relevant information for events nearby, a satellite image of their current location, and weather for the day. You can see my contributions below.

OpenWeather API

Clouds cleared up.

The OpenWeather API was used to fetch weather information for the user's location. The API was integrated into the site by using the OpenWeather API key. The API was used to pull up the current weather, the current weather icon, and the current weather description. It was originally pulling the weather in Kelvin, but I integrated a button that would change the temperature to either Fahrenheit or Celsius.

NASA API

Stars aligned.

Originally tasked to a different team member, due to extenuating circumstances I was given the task in the eleventh hour. After being able to fetch the image, it gave difficulties as it had no documentation for fixed sizes. Due to this, I figured out how to resize the image to a fixed size once it was called in and do so as well no matter the screen size.

Styling

Quickly Cohesive

The styling was performed with the MDBootstrap library. Originally, the site was having problems due to the location not being immediately available. Instead of performing logic with useEffect, I made a landing page instructing the user to click a button to confirm their agreement to gather their location data. After doing so, it'd take them to a page that held a card group with each API call. From there, each has a call to action button the user can easily identify. When clicking each button, they can they then see the details that we called in each API.

Otherwise, I crafted a muted navbar that showed the users GPS coordinates we gathered. Additionally, if they click the Google Maps icon, it'll take them to that coordinate within Google Maps itself. The background of the site was a GIF that was called with a mask over it so the site was easier on the eyes.

Giphy Go

Technologies used:

Where Gif's Can Be Gif's

A simple, straightforward website that allows users to search for gifs and save them to their computer. It showed another beginning step for my understanding on how API's are used. It shows the top result on the banner, where the next 6 results are displayed at the bottom of the page.

Giphy API

Capturing the motion.

Giphy's API is very versatile. It allows you to search for gifs, get trending gifs, get gifs by category, and get gifs by user. Obviously, my gif fetch is limited due to the scope of the project that would take less than a week to complete. My one big difficulty is how Giphy masks away their file types. They don't have a feature where you can download gif's directly, and obviously would like you to go to their page to do so, and I wasn't able to find a workaround for that.

Styling

Containing the rainbow.

To contain the fun of images and the general styling, I learned in detail about card, image flexing, and the grid system. I would definitely make a note of the grid system, as it is very important for the layout of the website. I would have changed some of the styling to make it more appealing and obvious for the fact that there is more content to be seen.

For third-party calls especially, reading documentation is vital for understanding how to pull what you are desiring to pull. Coming from a large background in technical documentation upkeep, being detail-oriented in your fetches are vital and I feel in sync with combining those skills.

Project Details for

All Ball Call

Technologies used:

P.E.R.N. Full Stack Web Application

Dev Don't Lie.

All Ball Call is a web application that is meant to become a platform to find any NBA highlight you can think up in a more intuitive and user friendly way than anywhere else. It's built with a PERN stack and TypeScript, with Tailwind 3.0 and Material UI on the frontend. It's meant to be the place where you can see the hottest highlights, yet find that one specific play you've always been looking for, and that aspect is the most work in progress. It's the project where I've certainly learned the most as I challenged myself with legacy React class components, picked up Tailwind, and TypeScript all in one go. Now that I believe I've conquered the aspect of class components, I am using this application to further my learning on more React Hooks.

Backend.

CRUD goals and the Admin.

Let's start with what I learned from this website so far. Firstly, my goal was to have at least three CRUD elements, including the user information. Within a few weeks, I was able to attain that with the user information, video posts, as well as comments. In addition, I added the ability to create an Admin account and have the ability to edit and delete nearly all content if necessary and add flags.

Class Components

Stepping back to go forward.

The challenge I gave to myself is in previous projects I dealt with only functional components. I wanted to understand how to use class components and their lifecycle methods to be able to adjust when necessary depending on future teams I join. This added versatility I feel will pay off for all the challenges this has given me attempting this. In future updates of my application, I am planning on now learning how to convert class components into functional components. Further utilization of more diverse amount of hooks where appropriate will be implemented as well.

Of course.

CORS denied my presentation.

A big struggle that I've continued to deal with is dealing with CORS flags. The irony is, within all my testing, I never ran into this issue as I've dealt with CORS in past projects and thought I scaled it correctly. In the end, with so much content being added, in the midst of my first technical demonstration I ran into the policy denying my content from appearing from my server side. Despite rigorous testing, it happened repeatedly and I had to show my local version temporarily. I've learned greatly from this presentation and within the code as well.

Project Details for

Waves: A Music App

Technologies used:

Open-Source Learning

Using a very small selection of open-source songs, I created a frontend music application with the smallest design details attempted and implemented. From the gradient progress bar, to the sidebar animations, I was able to create a very clean design.

Difficulties

The sidebar.

My biggest bug and roadblock I ran into was the sidebar. My struggle was in having it appropriately be responsive and able to have all the elements able to be interacted with whether on desktop or mobile. From there, I was able to have it become responsive both between the previous/next song button as well as tapping/clicking on the song. From there, I finally came to a resolution on the design.

Styling

Making my own progress (bar).

The progress bar was the challenge I was most proud to overcome and do so quickly. I had to develop a great understanding of this audio element CSS properties to be able to override the default styling they usually are given. I made it dynamic based on the song being played as well, using values from the artwork going from one color to the next.

Forwards and Back.

GSAP Animations.

This was my first time using the GSAP animation library. After reading over lots of their documentation, I definitely felt very much in control of how my UX would look and move. The learning curve was a lot easier than expected and I will be using this animation library in the future along with framer-motion.
